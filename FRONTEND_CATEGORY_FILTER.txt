========================================================
  CATEGORY FILTER — FRONTEND INTEGRATION GUIDE
========================================================

CONTEXT
-------
The product list and filters already work fine with all existing
params (colors, sizes, brands, price, organic, recycled, sortBy, etc.).
This guide only explains HOW TO ADD the category filtering
on top of what already works — nothing existing should change.


HOW CATEGORIES ARE STRUCTURED
-------------------------------
Categories have 2 levels:

  Parent category   →  e.g. "headwear-accessories"
    └── Subcategory →  e.g. "caps", "hats", "beanies"

Products are stored under subcategories.
When you filter by a parent categorySlug, the API automatically
includes ALL its subcategories — so you never get an empty result.


NEW QUERY PARAMS TO ADD
------------------------
Both endpoints accept these two new optional params:

  categorySlug      (string, optional)  — parent category slug
  subCategorySlug   (string, optional)  — subcategory slug
                                          only valid together with categorySlug


--------------------------------------------------------
  ENDPOINT 1: GET /products   (product list)
--------------------------------------------------------

Existing call (still works exactly the same):
  /products?lang=nl&colors=RED&sortBy=price_asc&page=1&limit=24

Add category on top:
  /products?lang=nl&categorySlug=headwear-accessories

Add subcategory on top:
  /products?lang=nl&categorySlug=headwear-accessories&subCategorySlug=caps

Combined with existing filters (nothing breaks):
  /products?lang=nl&categorySlug=headwear-accessories&colors=RED&sortBy=price_asc&page=1&limit=24


--------------------------------------------------------
  ENDPOINT 2: GET /products/filters   (filter panel)
--------------------------------------------------------

Existing call (still works exactly the same):
  /products/filters?lang=nl&colors=RED&minPrice=10

Add category on top:
  /products/filters?lang=nl&categorySlug=headwear-accessories

Add subcategory on top:
  /products/filters?lang=nl&categorySlug=headwear-accessories&subCategorySlug=caps

Combined with existing filters (nothing breaks):
  /products/filters?lang=nl&categorySlug=headwear-accessories&colors=RED&minPrice=10


--------------------------------------------------------
  RECOMMENDED FRONTEND FLOW
--------------------------------------------------------

1. User visits homepage / no category selected
   → call both endpoints WITHOUT categorySlug (existing behavior, no change)

2. User clicks a parent category (e.g. "headwear-accessories")
   → add &categorySlug=headwear-accessories to both calls
   → keep all other active filters (colors, price, etc.)

3. User clicks a subcategory (e.g. "caps")
   → keep &categorySlug=headwear-accessories
   → add &subCategorySlug=caps
   → keep all other active filters

4. User removes subcategory (goes back to parent)
   → remove subCategorySlug, keep categorySlug

5. User removes category entirely
   → remove both categorySlug and subCategorySlug


--------------------------------------------------------
  IMPORTANT RULES
--------------------------------------------------------

- subCategorySlug MUST always be sent together with categorySlug.
  Never send subCategorySlug alone — the API will reject it.

- If subCategorySlug does not belong to categorySlug,
  the API returns: 400 "Subcategory does not belong to this category"

- If categorySlug does not exist in the database,
  the API returns: 400 "Category slug not found"

- All existing filter params (colors, sizes, brands, minPrice, maxPrice,
  organic, recycled, sortBy, page, limit, q) continue to work
  exactly as before — adding categorySlug does not change them.
